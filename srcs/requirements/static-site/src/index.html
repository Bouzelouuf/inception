Inception Project - 42

Project Summary

The Inception project is an introduction to system administration, containerization, and DevOps best practices. You're tasked with building your own infrastructure using Docker and Docker Compose. Each component (Nginx, WordPress, MariaDB, Redis, etc.) runs in its own container, isolated and secure.

🛠️ Think of it like building your own digital house — every service is a room, and Docker Compose is your floor plan.

Nginx

Nginx is a web server and reverse proxy. It handles HTTPS connections and forwards user requests to the appropriate containers.

🔍 Analogy: Nginx is the receptionist that guides visitors to the right room while checking their ID (SSL certificate).

Redis

Redis is a fast, in-memory caching system. It stores temporary data like sessions or query results to reduce load on the database.

🔍 Analogy: Redis is a whiteboard with quick notes — instead of asking the database every time, we look here first.

Adminer

Adminer is a web interface to manage your MariaDB database. It helps you view, create, and delete tables or records.

🔍 Analogy: Adminer is the dashboard that lets you control and inspect your database without needing command-line tools.

FTP (vsftpd)

FTP allows file transfer between your machine and the server. In this project, it's configured with TLS for security.

🔍 Analogy: FTP is like a secure locker room where you can store and retrieve files safely.

About This Site

This is a static HTML site hosted by Nginx. It contains information about each service in the Inception project. It doesn’t use a database or backend — just HTML and CSS.

🔍 Analogy: It’s like a printed brochure – simple, fast, and easy to read.
